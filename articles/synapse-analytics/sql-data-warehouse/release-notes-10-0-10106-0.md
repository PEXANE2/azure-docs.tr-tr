# <a name="azure-synapse-analytics-release-notes"></a>Azure SYNAPSE Analytics sürüm notları

Bu makalede, Azure SYNAPSE 'de [SYNAPSE SQL](sql-data-warehouse-overview-what-is.md) 'in son sürümlerindeki yeni özellikler ve geliştirmeler özetlenmektedir. Makalede ayrıca, yayınla doğrudan ilgili olmayan ancak aynı zaman çerçevesinde yayınlanan önemli içerik güncelleştirmeleri de listelenir. Diğer Azure hizmetlerine yönelik iyileştirmeler için bkz. [hizmet güncelleştirmeleri](https://azure.microsoft.com/updates).

## <a name="check-your-azure-synapse-version"></a>Azure SYNAPSE sürümünüzü denetleyin

Yeni özellikler tüm bölgelere alındığı için, örneğinize dağıtılan sürümü ve özellik kullanılabilirliği için en son sürüm notlarını kontrol edin. Sürümü denetlemek için SQL Server Management Studio (SSMS) aracılığıyla SQL havuzunuza bağlanın ve geçerli sürümü döndürmek için `SELECT @@VERSION;` komutunu çalıştırın.

SQL havuzunuza hangi sürümün uygulandığını onaylamak için tanımlanan sürümü kullanın. Çıkışdaki Tarih, SQL havuzunuza uygulanan yayın için ayı belirler.

> [!NOTE]
> SELECT @@VERSION tarafından döndürülen ürün adı Microsoft Azure SQL veri ambarı 'Den Azure SYNAPSE Analytics 'e değişecektir. Değişiklik yapılmadan önce gelişmiş bildirim göndereceğiz. Bu değişiklik, uygulama kodundaki @@VERSION öğesini seçme sonucundan ürün adını ayrıştırlayan müşteriler için uygundur. Ürün remarkalaması nedeniyle uygulama kodu değişikliklerinden kaçınmak için lütfen şu komutları kullanarak veritabanı ürün adı ve sürümü için SERVERPROPERTY 'yi sorgulayın: XX sürüm numarasını döndürecek şekilde. X. XXXXX. X (ürün adı olmadan) şu komutu kullanın:
>
> ```sql
> SELECT SERVERPROPERTY('ProductVersion')
>
> --To return engine edition, use this command that returns 6 for Azure Synapse Analytics (Formerly SQL Data Warehouse):
>
> SELECT SERVERPROPERTY('EngineEdition')
> ```

Müşteriler, 100 cDWU ([veri ambarı birimleri](what-is-a-data-warehouse-unit-dwu-cdwu.md)) Ile başlayan Azure SYNAPSE 'ın önde gelen performans, esneklik ve güvenlik özellikleriyle karşılaşabilirler ve dakikalar Içinde 30.000 cdwu 'e ölçeklendirebilir. Müşteriler, 2018 Aralık ' ten itibaren, [bölgelerde](gen2-migration-schedule.md#automated-schedule-and-region-availability-table)daha düşük işlem katmanlarıyla Gen2 performans ve esnekliğinden yararlanarak, diğer bölgeler 2019 sırasında kullanılabilir.<br/><br/>Microsoft, yeni nesil veri ambarına yönelik giriş noktasını bırakarak, güvenli, yüksek performanslı bir veri ambarının tüm avantajlarını tahmin etmeden değerlendirmek isteyen, değer odaklı müşterilere olan kapıların her birini kendileri için en iyi şekilde değerlendirmenizi sağlar. Müşteriler, geçerli 500 cDWU giriş noktasından düşük 100 cDWU olarak başlayabilir. SQL Analytics, işlemleri duraklatma ve sürdürmeyi desteklemeye devam eder ve yalnızca işlem esnekliğinden daha fazla ilerler. Gen2 Ayrıca sınırsız sütun depolama kapasitesini, sorgu başına en fazla 128 eşzamanlı sorguya ve [Uyarlamalı önbelleğe alma](https://azure.microsoft.com/blog/adaptive-caching-powers-azure-sql-data-warehouse-performance-gains/) özelliklerine kadar 2,5 kat daha fazla bellek ile destekler. Ortalamanın bu özellikleri, Gen1 'deki aynı veri ambarı birimiyle aynı fiyata kıyasla beş kat daha fazla performans sağlar. Coğrafi olarak yedekli yedeklemeler, yerleşik garantili veri korumasına sahip Gen2 için standarttır. Azure 'da SQL Analytics, siz olduğunuzda ölçeklendirmeye SYNAPSE.|
|**Columnstore arka plan birleştirme**|Varsayılan olarak, Azure SQL verileri, [RowGroups](sql-data-warehouse-memory-optimizations-for-columnstore-compression.md)adlı mikro bölümlerle verileri sütunlu biçimde depolar. Bazen, dizin oluşturma veya veri yükleme sırasındaki bellek kısıtları nedeniyle RowGroups, 1.000.000 satırlık en uygun boyuttan daha az olacak şekilde sıkıştırılabilir. RowGroups, silme nedeniyle de parçalanabilir. Küçük veya parçalanmış RowGroups, daha yüksek bellek tüketimine ve verimsiz sorgu yürütmeye neden olur. Bu sürümle birlikte, columnstore arka plan bakım görevi, belleği daha iyi hale getireve sorgu yürütmeyi hızlandırmaya yönelik daha büyük RowGroups oluşturmak için küçük sıkıştırılmış RowGroups birleştirir.
